# 07. 파이썬컴퓨팅: 모듈과 패키지

## 학습 목표
1. 모듈(Module)의 개념을 이해하고, 코드를 여러 파일로 분리하여 관리하는 필요성을 설명할 수 있다.
2. `import`, `from ... import` 등 다양한 방법으로 모듈을 가져와 사용하는 방법을 학습한다.
3. 패키지(Package)를 활용하여 관련된 모듈들을 계층적인 구조로 구성하고 관리하는 능력을 함양한다.

---

## 📝 요약 (Summary)
1.  **모듈 (Module)**: 함수, 변수, 클래스를 모아놓은 파이썬 파일(`.py`)입니다. 코드의 **재사용성**을 높이고, 관련된 코드들을 하나의 파일로 묶어 **관리**를 용이하게 합니다.
2.  **패키지 (Package)**: 점(.)으로 구분된 계층적 모듈 이름 공간을 의미합니다. 여러 관련 모듈들을 디렉터리 구조로 묶어 관리하는 방법으로, 대규모 프로젝트에서 코드의 구조를 체계적으로 유지하는 데 필수적입니다.
3.  **모듈 가져오기 (Import)**: 다른 파일(모듈)에 정의된 기능을 현재 파일에서 사용하기 위한 명령어입니다. `import 모듈명`은 모듈 전체를, `from 모듈명 import 함수명`은 특정 기능만을 가져옵니다. `as` 키워드로 별칭을 지정할 수 있습니다.
4.  **`if __name__ == "__main__":`**: 스크립트 파일이 직접 실행될 때만 특정 코드 블록이 동작하도록 만드는 파이썬의 관용구입니다. 이를 통해 하나의 파일을 **직접 실행 가능한 스크립트**이면서 동시에 다른 파일에서 **가져올 수 있는 모듈**로 활용할 수 있습니다.

## ✏️ 심화 학습 (Study Subject)
1.  **가상 시나리오: "나만의 미니 계산기 라이브러리 만들기"**
    - **상황**: 여러 프로젝트에서 공통적으로 사용될 사칙연산 및 고급 연산(팩토리얼, 제곱) 기능이 필요해졌습니다. 이 기능들을 매번 복사-붙여넣기 하는 것은 비효율적입니다.
    - **해결**: 연산 기능들을 별도의 모듈과 패키지로 구성하여 '라이브러리'처럼 만듭니다.
      1.  `my_calculator` 디렉터리(패키지)를 생성합니다.
      2.  기본 연산을 담은 `basic_ops.py` 모듈과 고급 연산을 담은 `advanced_ops.py` 모듈을 만듭니다.
      3.  `my_calculator` 디렉터리 안에 `__init__.py` 파일을 만들어 패키지로 인식시킵니다. 이 파일에서 `from .basic_ops import add, subtract` 와 같이 하위 모듈의 함수를 미리 가져와 사용 편의성을 높일 수 있습니다.
      ```
      # 사용 예시
      from my_calculator import basic_ops, advanced_ops
      
      print(basic_ops.add(10, 5))        # 15
      print(advanced_ops.factorial(5)) # 120
      ```
    - **AI 학습 포인트**: AI 학습 도우미에게 "위 `my_calculator` 패키지를 `pip`으로 설치하고 제거할 수 있는 패키지처럼 만들려면 어떤 파일(`setup.py` 등)이 추가로 필요한가요?"라고 질문하여 파이썬 패키지 배포의 기초를 학습할 수 있습니다.

2.  **관점별 비교 및 오개념 분석: `import A` vs. `from A import B`**
    - **핵심 비교**: 이름 공간(Namespace)의 차이
        - **`import A`**: 모듈 `A` 자체를 현재 이름 공간으로 가져옵니다. `A` 안의 함수 `B`를 사용하려면 항상 `A.B()`와 같이 모듈 이름을 접두사로 붙여야 합니다. 이름 충돌의 위험이 적고, 코드의 출처가 명확합니다.
        - **`from A import B`**: 모듈 `A`에서 `B`라는 이름(함수, 변수 등)만 직접 현재 이름 공간으로 가져옵니다. 사용할 때 `B()`처럼 바로 호출할 수 있어 편리하지만, 현재 코드에 이미 `B`라는 이름이 있다면 덮어쓰게 되어 이름 충돌(Name Collision)이 발생할 수 있습니다.
    - **오개념 1: `from A import *`는 편리하므로 자주 사용해도 괜찮다?**
        - **분석**: `*`를 사용하면 해당 모듈의 모든 공개된 이름들을 현재 이름 공간으로 가져옵니다. 이는 매우 편리해 보이지만, 어떤 이름들이 들어왔는지 알기 어렵고, 자신도 모르게 기존의 변수나 함수를 덮어쓸 위험이 매우 큽니다. 이는 코드의 가독성과 예측 가능성을 심각하게 해치므로, PEP 8(파이썬 스타일 가이드)에서는 `__all__`을 명시한 특별한 경우를 제외하고는 사용을 지양하도록 권고합니다.
    - **오개념 2: 모듈을 `import`하면 파일의 모든 코드가 매번 다시 실행된다?**
        - **분석**: 파이썬은 성능을 위해 한 번 `import`된 모듈을 `sys.modules`라는 딕셔너리에 캐싱(caching)합니다. 동일한 모듈을 다시 `import`할 때는 파일을 다시 읽고 실행하는 대신, 캐시된 모듈 객체를 즉시 반환합니다. 따라서 모듈의 최상위 레벨에 있는 코드는 최초 `import` 시에 단 한 번만 실행됩니다. (만약 강제로 다시 로드하고 싶다면 `importlib.reload()` 함수를 사용해야 합니다.)
