# 02. 데이터의 관리와 분석

## 학습 목표
1. 데이터 수집 방법을 이해할 수 있다.
2. 데이터베이스의 개념과 중요성을 설명할 수 있다.
3. 데이터베이스 관리 시스템에 대해 이해할 수 있다.
4. 데이터 분석의 의의를 설명할 수 있다.
5. 데이터 분석의 핵심문제를 파악할 수 있다.
6. 데이터 분석기회를 구조화하고 구체화할 수 있다.

---

## 📝 요약 (Summary)
1.  **데이터 수집(Data Collection)**: 분석의 첫 단계로, 내부 시스템, 웹 크롤링/스크래핑, 공공데이터 API 등 다양한 경로를 통해 필요한 데이터를 확보하는 과정입니다.
2.  **데이터베이스(Database)**: 여러 사람이 공유하고 사용할 목적으로, 논리적으로 연관된 데이터를 **구조화**하여 통합 관리하는 정보의 집합입니다. 데이터의 중복을 최소화하고 검색 및 갱신 효율을 높이는 것이 목적입니다.
3.  **데이터베이스 관리 시스템(DBMS)**: 사용자가 데이터베이스를 효율적으로 생성, 조회, 수정, 삭제(CRUD)할 수 있도록 돕는 소프트웨어 시스템입니다. 데이터의 무결성, 보안, 동시성 제어 등의 기능을 제공합니다.
4.  **데이터 모델(Data Model)**: 데이터의 논리적 구조와 데이터 간의 관계를 표현하는 방법입니다. **관계형 모델**은 데이터를 2차원 테이블 형태로 표현하며, 현재 가장 널리 사용됩니다.
5.  **데이터 분석 기회 포착**: 성공적인 분석은 기술이 아닌 '올바른 질문'에서 시작됩니다. 비즈니스 문제를 데이터 분석 문제로 전환하고, 분석의 목표를 명확히 정의하는 과정이 중요합니다.
6.  **분석 문제 구체화**: '매출 증대'와 같은 추상적인 목표를 '신규 고객의 분기별 재구매율 20% 향상'처럼 구체적이고 측정 가능한 **핵심 성과 지표(KPI)**로 변환하여 분석의 방향을 명확히 하고 성공 여부를 판단하는 기준을 마련하는 과정입니다.

## ✏️ 심화 학습 (Study Subject)

### 1. 데이터 수집 방법
데이터 분석의 첫걸음은 신뢰할 수 있는 데이터를 확보하는 것입니다. 데이터의 원천은 크게 내부 데이터와 외부 데이터로 나눌 수 있으며, 다음과 같은 다양한 경로를 통해 수집할 수 있습니다.

1.  **내부 데이터 활용**:
    -   **설명**: 기업이나 기관이 자체적으로 운영하는 시스템(ERP, CRM, POS 등)에 축적된 정형 데이터를 활용하는 가장 기본적인 방법입니다.
    -   **예시**: 고객 정보, 거래 내역, 재고 데이터, 웹사이트 로그.

2.  **웹 크롤링 및 스크래핑**:
    -   **웹 크롤링(Web Crawling)**: 자동화된 로봇(크롤러)이 인터넷상의 여러 웹 페이지를 순회하며 URL과 같은 정보를 수집하는 기술입니다. 주로 검색 엔진이 웹 페이지를 수집하는 데 사용합니다.
    -   **웹 스크래핑(Web Scraping)**: 특정 웹 페이지에 접속하여, 그중에서 필요한 데이터(예: 상품 가격, 뉴스 기사, 주식 정보)만을 선택적으로 추출하는 기술입니다.
    -   **주의사항**: 무분별한 크롤링/스크래핑은 대상 서버에 부하를 줄 수 있으며, 사이트의 이용 약관(robots.txt)을 반드시 준수해야 합니다.

3.  **API (Application Programming Interface) 활용**:
    -   **설명**: 다른 서비스나 플랫폼이 제공하는 기능을 외부에서 쉽게 호출하고 정제된 데이터를 얻을 수 있도록 만들어진 통신 규약입니다.
    -   **예시**:
        -   **공공데이터포털(data.go.kr)**: 정부 기관이 보유한 날씨, 교통, 인구 등의 데이터를 API 형태로 제공받아 활용할 수 있습니다.
        -   **소셜 미디어 API**: 트위터, 페이스북 등이 제공하는 API를 통해 특정 키워드에 대한 여론이나 트렌드를 분석할 수 있습니다.

4.  **IoT 센서 및 로그 데이터**:
    -   **설명**: 스마트 팩토리의 기계 센서, 스마트 워치의 건강 데이터, 모바일 앱 사용 로그 등 사물인터넷(IoT) 기기나 다양한 시스템에서 실시간으로 생성되는 데이터를 수집합니다.

### 2. 데이터베이스와 DBMS

#### 가. 데이터베이스 (Database, DB)
-   **정의**: 여러 사람이 공유하고 사용할 목적으로, 논리적으로 연관된 데이터를 **구조화**하여 통합 관리하는 정보의 집합입니다.
-   **목적**: 데이터의 중복을 최소화하고, 데이터 무결성을 유지하며, 검색과 갱신(수정, 삭제)의 효율성을 높이는 데 있습니다. 단순히 데이터를 모아놓은 파일 시스템과 달리, 체계적인 관리를 통해 데이터의 가치를 극대화합니다.
-   **비유**: 파일 시스템이 책을 방바닥에 아무렇게나 쌓아두는 것이라면, 데이터베이스는 도서관의 서가처럼 특정 규칙(장르, 저자 등)에 따라 책을 체계적으로 정리하고 색인(Index)을 만들어 누구나 쉽게 원하는 책을 찾을 수 있도록 하는 것과 같습니다.

#### 나. 데이터베이스 관리 시스템 (Database Management System, DBMS)
-   **정의**: 사용자가 데이터베이스에 쉽게 접근하고 이를 효율적으로 관리(생성, 조회, 수정, 삭제 등)할 수 있도록 돕는 소프트웨어 시스템입니다.
-   **주요 기능**:
    -   **정의(Definition)**: 데이터의 구조, 타입, 제약 조건 등을 정의합니다.
    -   **조작(Manipulation)**: 데이터를 검색, 삽입, 수정, 삭제하는 기능(CRUD: Create, Read, Update, Delete)을 제공합니다.
    -   **제어(Control)**: 데이터의 **무결성**(정확성 유지), **보안**(접근 제어), **동시성 제어**(여러 사용자가 동시에 접근할 때 발생하는 문제 제어) 등을 관리합니다.
-   **장점**:
    -   **데이터 독립성 확보**: 물리적 저장 구조나 논리적 구조가 변경되어도 응용 프로그램은 영향을 받지 않습니다.
    -   **데이터 중복 최소화 및 일관성 유지**: 데이터를 통합 관리하여 중복을 줄이고, 한 곳에서 수정하면 모든 곳에 반영되어 일관성을 유지합니다.
    -   **보안 강화**: 사용자별로 접근 권한을 차등 부여하여 데이터를 안전하게 보호합니다.

#### 다. 데이터 모델 (Data Model)
데이터 모델은 현실 세계의 정보를 컴퓨터가 이해하고 처리할 수 있는 형태로 변환하기 위해, 데이터의 구조와 관계, 제약 조건 등을 논리적으로 표현하는 방법입니다.

| 구분 | 계층형 모델 (Hierarchical) | 네트워크형 모델 (Network) | 관계형 모델 (Relational) |
| :--- | :--- | :--- | :--- |
| **구조** | 트리(Tree) 구조 | 그래프(Graph) 구조 | 2차원 테이블(Table) 구조 |
| **관계** | 1:N (부모-자식) | N:M (다대다) | 테이블 간의 관계 (Key 사용) |
| **특징** | 구조가 단순, 구현 용이 | 복잡한 관계 표현 가능 | 구조가 유연하고 이해하기 쉬움 |
| **단점** | 복잡한 관계 표현 어려움 | 구조가 복잡, 유지보수 어려움 | 성능 저하 가능성 (Join 연산) |
| **예시** | 컴퓨터 파일 시스템, 조직도 | 수강신청 시스템 | **MySQL, Oracle, PostgreSQL** 등 |

### 3. 데이터 분석 기회의 포착과 구체화
성공적인 데이터 분석은 뛰어난 기술이 아니라, **'올바른 질문'**에서 시작됩니다. 비즈니스 가치를 창출하기 위해서는 막연한 비즈니스 문제를 구체적이고 실행 가능한 데이터 분석 문제로 전환하는 과정이 필수적입니다.

#### 가. 분석 기회 포착 접근법
분석 기회란 '주요 의사결정이 필요한 업무를 정의하고, 이를 효과적으로 수행하기 위해 알아야 할 사항들을 데이터 분석 관점에서 정리하고 탐색하는 과정'입니다. 접근 방식은 크게 하향식과 상향식으로 나뉩니다.

| 구분 | 하향식 접근 (Top-down) | 상향식 접근 (Bottom-up) |
| :--- | :--- | :--- |
| **출발점** | **비즈니스 문제**에서 출발 | **데이터**에서 출발 |
| **핵심 질문** | "우리 회사의 가장 큰 문제는 무엇인가?"<br>"이 문제를 해결하려면 무엇을 알아야 하는가?" | "우리가 가진 데이터로 무엇을 할 수 있을까?"<br>"이 데이터에서 흥미로운 패턴은 무엇인가?" |
| **과정** | 문제 정의 → 가설 수립 → 필요 데이터 정의 및 분석 | 데이터 탐색 → 패턴/인사이트 발견 → 비즈니스 문제와 연결 |
| **장점** | 분석 목표가 명확하고 비즈니스 기여도가 높음 | 예상치 못한 새로운 기회(인사이트) 발견 가능 |
| **단점** | 기존의 틀에 갇혀 새로운 시각을 놓칠 수 있음 | 분석 결과가 실제 비즈니스 문제와 연결되지 않을 수 있음 |

#### 나. 분석 문제 구체화 기법
막연한 문제를 구체적인 분석 과제로 만들기 위해 다음과 같은 기법을 사용할 수 있습니다.

1.  **연속 질문 방식 (5 Whys)**: "왜?"라는 질문을 5번 반복하여 문제의 근본 원인을 파고드는 방법입니다.
    -   **문제**: 고객 이탈률이 높다.
    -   **Why 1?** 신규 고객의 재방문율이 낮기 때문이다.
    -   **Why 2?** 첫 구매 경험에 만족하지 못했기 때문이다.
    -   **Why 3?** 배송이 지연되는 경우가 많았기 때문이다.
    -   **Why 4?** 특정 지역으로 가는 물류 처리 과정에 병목 현상이 있기 때문이다.
    -   **Why 5?** 해당 지역의 물류 파트너사 시스템 연동에 오류가 있기 때문이다.
    -   **→ 분석 과제**: '특정 지역의 배송 지연 데이터와 물류 파트너사 시스템 로그 분석'

2.  **업무 프로세스 분해**: 특정 업무 프로세스(예: 마케팅 → 판매 → 고객 관리)를 단계별로 나누고, 각 단계의 목표, 투입 자원, 성과 지표 등을 질문하며 분석 과제를 도출합니다.

#### 다. 목표 가치의 구체화 (KPI 설정)
분석의 목표는 반드시 **측정 가능**해야 합니다. 추상적인 목표를 구체적인 **핵심 성과 지표(KPI, Key Performance Indicator)**로 변환해야 분석의 성공 여부를 객관적으로 판단할 수 있습니다.

| 구분 | Before (추상적 목표) | After (구체적 KPI) |
| :--- | :--- | :--- |
| **목표** | "매출을 늘리고 싶다." | "다음 분기 신규 고객의 재구매율을 20% 향상시킨다." |
| **측정** | 불가능 | 가능 (재구매율 측정) |
| **판단 기준** | 모호함 | 명확함 (20% 달성 여부) |

##### [AI 학습 강화] 가상 시나리오: 온라인 쇼핑몰의 고객 이탈 문제 해결
> **상황**: 당신은 온라인 쇼핑몰의 데이터 분석가입니다. CEO가 "최근 고객들이 우리 서비스를 떠나는 것 같아요. 원인을 분석하고 해결책을 찾아주세요."라는 막연한 문제를 제시했습니다.

1.  **문제 구체화 (하향식 접근)**
    -   '고객 이탈'을 '최근 6개월간 구매 이력이 없는 휴면 전환 고객'으로 정의합니다.
    -   **가설 1**: 신규 고객의 첫 구매 경험이 나쁘기 때문일 것이다.
    -   **가설 2**: 경쟁사의 공격적인 마케팅으로 고객을 빼앗겼기 때문일 것이다.

2.  **분석 과제 도출**
    -   **(가설 1 검증)** '첫 구매 후 30일 내 재구매율'과 '첫 구매 상품 카테고리' 간의 관계를 분석합니다. 또한, '첫 구매 시 배송 만족도 평점'과 '휴면 전환율'의 상관관계를 분석합니다.
    -   **(가설 2 검증)** '경쟁사 할인 이벤트 기간'과 '자사 앱의 일일 활성 사용자(DAU) 감소' 간의 관계를 분석합니다.

3.  **분석 및 결과 해석**
    -   분석 결과, '뷰티' 카테고리 상품을 처음 구매한 고객의 휴면 전환율이 유독 높았고, 이들은 대부분 배송 만족도에 낮은 평점을 준 것으로 나타났습니다.

4.  **해결 방안(액션 아이템) 제시**
    -   **단기**: '뷰티' 카테고리 상품의 포장 및 배송 프로세스를 점검하고 개선합니다.
    -   **장기**: 첫 구매 고객에게 배송 현황 알림을 강화하고, 배송 완료 후 만족도 조사를 통해 부정적 경험을 조기에 파악하여 대응하는 시스템을 구축합니다.

<a id="prob-2"></a>

> **정답: ② 웹 스크래핑(Web Scraping)**

<a id="prob-4"></a>

<a id="prob-5"></a>
**5. 다음 중 데이터베이스 관리 시스템(DBMS)을 사용하는 가장 큰 장점으로 옳지 않은 것은?**

## 📖 심화 학습 예시 답안
#### 1. 데이터의 기본 구조에 대해 정리해 보시오.
데이터의 기본 구조는 데이터를 구성하는 가장 작은 단위부터 시작하여 점차 큰 집합으로 확장되는 계층적 구조를 가집니다.
1.  **비트(Bit)**: 컴퓨터가 처리하는 정보의 최소 단위. 0 또는 1의 값을 가집니다.
2.  **바이트(Byte)**: 8개의 비트가 모여 구성된 단위. 하나의 문자(영문, 숫자 등)를 표현할 수 있습니다.
3.  **필드(Field)**: 하나 이상의 바이트가 모여 의미 있는 정보를 나타내는 최소 단위. 속성(Attribute) 또는 열(Column)이라고도 합니다. (예: '이름' 필드, '나이' 필드)
4.  **레코드(Record)**: 관련된 필드들의 집합. 하나의 개체(Entity)에 대한 정보를 담고 있으며, 튜플(Tuple) 또는 행(Row)이라고도 합니다. (예: 한 학생의 이름, 학번, 학과로 구성된 정보)
5.  **파일(File)**: 동일한 구조의 레코드들의 집합. 테이블(Table)이라고도 합니다. (예: 전체 학생 정보 파일)
6.  **데이터베이스(Database)**: 논리적으로 연관된 하나 이상의 파일(테이블)들의 집합. 데이터의 중복을 최소화하고 공유를 목적으로 체계적으로 관리됩니다.

#### 2. 데이터의 수집 방법을 나열하고 그중 한가지 방법을 택해 데이터를 수집하고 정리해 보시오.
**데이터 수집 방법**:
- **내부 데이터 활용**: 기업 내 CRM, ERP, 판매 시스템(POS) 등에 축적된 데이터 활용
- **웹 크롤링/스크래핑**: 웹사이트에서 필요한 정보를 자동화된 프로그램으로 수집
- **API(Application Programming Interface) 활용**: 공공데이터포털, 소셜 미디어 등에서 제공하는 API를 통해 정형화된 데이터 수집
- **설문조사/실험**: 특정 목적을 위해 설문지를 배포하거나 통제된 환경에서 실험을 통해 데이터 생성
- **IoT 센서 데이터 수집**: 스마트 팩토리, 스마트 홈 등의 센서로부터 실시간 데이터 수집

**사례: 공공데이터 API를 활용한 '서울시 전기차 충전소 정보' 수집**
1.  **방법 선택**: 공공데이터포털(data.go.kr)에서 제공하는 API를 활용하기로 결정.
2.  **API 신청 및 인증키 발급**: '서울시 전기차 충전소 정보' API를 검색하여 활용 신청을 하고, 발급받은 일반 인증키를 확인.
3.  **데이터 수집 (Python 코드 예시)**:
    ```python
    import requests
    import pandas as pd

    API_KEY = "YOUR_API_KEY" # 발급받은 인증키
    URL = f"http://openapi.seoul.go.kr:8088/{API_KEY}/json/evChargerStation/1/100/"

    # API 요청 및 데이터 수신
    response = requests.get(URL)
    data = response.json()

    # 필요한 정보 추출 및 정리
    stations = data['evChargerStation']['row']
    df = pd.DataFrame(stations)

    # 데이터 프레임 확인
    print(df[['STAT_NM', 'ADDR', 'USE_TIME', 'STAT_STAT']].head())
    ```
4.  **결과**: 위 코드를 실행하면 서울시 전기차 충전소 100개의 정보(충전소명, 주소, 이용가능시간, 충전기상태 등)가 Pandas 데이터프레임 형태로 깔끔하게 정리됩니다.

#### 3. 데이터 모델의 종류를 나열하고 각 모델의 예를 들어보시오.
데이터 모델은 데이터베이스의 논리적 구조를 표현하는 방식으로, 시대의 흐름에 따라 발전해왔습니다.
1.  **계층형 데이터 모델 (Hierarchical Model)**:
    - **설명**: 데이터를 부모-자식 관계의 트리(Tree) 구조로 표현합니다. 하나의 자식은 반드시 하나의 부모만 가질 수 있습니다 (1:N 관계).
    - **예시**: 기업의 조직도 (사장 - 부서장 - 팀원), 컴퓨터의 파일 시스템 폴더 구조.
2.  **네트워크형 데이터 모델 (Network Model)**:
    - **설명**: 데이터 구조를 그래프(Graph) 형태로 표현하여, 하나의 자식이 여러 부모를 가질 수 있습니다 (N:M 관계). 계층형 모델보다 복잡한 관계 표현이 가능합니다.
    - **예시**: 수강신청 시스템 (한 명의 학생은 여러 과목을 수강하고, 하나의 과목은 여러 학생이 수강).
3.  **관계형 데이터 모델 (Relational Model)**:
    - **설명**: 현재 가장 널리 사용되는 모델로, 데이터를 행(Row)과 열(Column)으로 구성된 2차원 테이블(Table 또는 Relation) 형태로 표현합니다. 테이블 간의 관계는 기본키(Primary Key)와 외래키(Foreign Key)를 사용하여 정의합니다.
    - **예시**: 대부분의 상용/오픈소스 데이터베이스 (Oracle, MySQL, PostgreSQL). '학생' 테이블과 '학과' 테이블이 '학과코드'라는 키로 연결된 구조.
4.  **객체지향 데이터 모델 (Object-Oriented Model)**:
    - **설명**: 프로그래밍의 객체지향 개념을 도입하여, 데이터와 데이터를 처리하는 메소드(Method)를 하나의 객체(Object)로 묶어서 관리합니다.
    - **예시**: 멀티미디어 데이터(이미지, 음성)나 CAD 데이터처럼 복잡한 구조의 데이터를 저장하는 데 사용됩니다.
5.  **NoSQL 데이터 모델 (Not Only SQL Model)**:
    - **설명**: 관계형 모델과 달리 고정된 스키마가 없는 유연한 데이터 모델을 사용합니다. 대용량 비정형 데이터 처리에 유리하며, 다양한 모델이 존재합니다.
    - **예시**:
        - **Key-Value 모델**: `Redis`, `Amazon DynamoDB`
        - **Document 모델**: `MongoDB`
        - **Graph 모델**: `Neo4j`

#### 4. 기업이나 기관의 데이터 분석에 있어서 분석의 대상 문제를 적절하게 정의하고 포착하기 위해 필요한 접근 방식에 대해 설명하시오.
분석 대상 문제를 적절하게 정의하고 포착하기 위해서는 기술 중심이 아닌 **비즈니스 문제 중심의 접근**이 필요하며, 다음과 같은 접근 방식을 조합하여 사용할 수 있습니다.

1.  **하향식 접근 (Top-down Approach)**
    -   **과정**: 비즈니스의 핵심 문제(예: '고객 이탈률 증가')에서 출발하여, 이를 해결하기 위해 필요한 질문들을 구체화하고, 그 질문에 답하기 위해 어떤 데이터가 필요한지를 역으로 찾아 나가는 방식입니다.
    -   **장점**: 분석의 목표가 명확하고, 분석 결과가 실제 비즈니스 가치로 직결될 가능성이 높습니다.

2.  **상향식 접근 (Bottom-up Approach)**
    -   **과정**: 현재 보유한 데이터를 먼저 탐색하며 이전에 알지 못했던 의미 있는 패턴이나 인사이트를 발견한 후, 이를 비즈니스 문제 해결에 어떻게 활용할 수 있을지를 모색하는 방식입니다.
    -   **장점**: 예상치 못한 새로운 분석 기회를 발견할 수 있습니다.

3.  **업무 프로세스 기반 분석 기회 발굴**
    -   **과정**: 실제 업무 흐름(예: 마케팅 → 판매 → 고객 관리)을 상세히 분석하고, 각 단계에서 발생하는 비효율이나 개선점을 파악하여 데이터 분석 과제로 정의합니다.
    -   **예시**: '고객이 장바구니에 상품을 담고도 최종 결제를 하지 않는 비율이 높다'는 문제점을 포착하고, '장바구니 이탈 고객의 행동 패턴 분석'이라는 과제를 정의할 수 있습니다.

가장 이상적인 방식은 Top-down 접근으로 분석의 큰 방향을 설정하고, Bottom-up 접근과 업무 프로세스 분석을 통해 구체적인 분석 아이디어를 보완하는 것입니다.

## 🌐 최신 동향 및 추가 정보

### 1. 데이터 메시 (Data Mesh)
데이터 메시(Data Mesh)는 중앙 집중화된 데이터 플랫폼(데이터 웨어하우스, 데이터 레이크)의 한계를 극복하기 위한 새로운 아키텍처 패러다임입니다. 각 비즈니스 도메인(예: 마케팅팀, 재무팀)이 해당 도메인의 데이터를 직접 소유하고, 분석에 용이한 '데이터 상품(Data Product)' 형태로 제공하여 다른 팀이 쉽게 사용할 수 있도록 하는 분산형 구조를 지향합니다. 이는 데이터의 민주화와 조직의 민첩성을 높이는 데 기여합니다.

### 2. 데이터 패브릭 (Data Fabric)
데이터 패브릭(Data Fabric)은 조직 내외부에 분산된 다양한 데이터 소스를 지능적으로 통합하고 연결하여, 사용자가 데이터의 물리적 위치나 구조에 상관없이 필요한 데이터에 원활하게 접근하고 분석할 수 있도록 지원하는 통합 데이터 관리 아키텍처입니다. 메타데이터, AI/ML 기술을 활용하여 데이터 검색, 거버넌스, 통합 등의 과정을 자동화하는 데 중점을 둡니다.

### 3. 클라우드 데이터베이스 (Cloud Database)
과거에는 기업이 자체 서버에 데이터베이스를 구축(On-premise)하는 것이 일반적이었지만, 현재는 AWS(Amazon Web Services), Google Cloud, MS Azure와 같은 클라우드 서비스 제공업체가 제공하는 데이터베이스를 구독 형태로 사용하는 것이 대세가 되었습니다. 클라우드 데이터베이스는 초기 구축 비용이 낮고, 필요에 따라 컴퓨팅 자원을 유연하게 확장/축소할 수 있으며, 안정적인 운영과 보안을 서비스 제공업체가 책임져준다는 장점이 있습니다.

## 📚 핵심 용어집 (Glossary)

- **데이터 모델 (Data Model)**: 현실 세계의 정보를 컴퓨터가 이해할 수 있도록 데이터의 구조, 관계, 제약 조건 등을 논리적으로 표현하는 방법. (예: 관계형 모델)
- **데이터베이스 (Database, DB)**: 여러 사용자가 공유하고 사용할 목적으로 통합하여 관리하는 구조화된 데이터의 집합.
- **데이터베이스 관리 시스템 (DBMS)**: 사용자와 데이터베이스 사이에서 데이터의 저장, 수정, 검색, 보안 등을 관리하는 소프트웨어. (예: MySQL, Oracle)
- **웹 스크래핑 (Web Scraping)**: 웹 페이지의 HTML 소스 코드에서 특정 데이터를 추출하는 기술.
- **웹 크롤링 (Web Crawling)**: 자동화된 봇(크롤러)이 하이퍼링크를 따라 이동하며 웹 페이지를 탐색하고 URL 목록을 수집하는 기술.
- **유저 스토리 (User Story)**: 사용자의 관점에서 시스템이 제공해야 할 가치를 간단한 형식으로 표현한 요구사항 명세.
- **정규화 (Normalization)**: 관계형 데이터베이스에서 데이터의 중복을 최소화하고 무결성을 보장하기 위해 테이블을 구조화하는 과정.
- **API (Application Programming Interface)**: 응용 프로그램이 서로 상호작용하고 데이터를 교환할 수 있도록 미리 정의된 규칙과 도구의 집합.
- **SQL (Structured Query Language)**: 관계형 데이터베이스에서 데이터를 정의, 조작, 제어하기 위해 사용하는 표준 질의 언어.

### 1. 가상 시나리오: "신규 서비스 개발을 위한 데이터베이스 선택"

> **상황**: 당신은 새로운 소셜 미디어 서비스를 개발하는 스타트업의 기술 리더입니다. 이 서비스는 사용자들이 텍스트, 이미지, 짧은 동영상을 공유하고, 친구 관계를 맺으며, '좋아요'를 누르는 기능을 가집니다. 서비스 초기에는 사용자가 적겠지만, 빠르게 성장할 가능성을 염두에 두어야 합니다. 어떤 종류의 데이터베이스를 선택해야 할까요?

**Q&A 기반 문제 해결 과정**
- **Q1. 데이터베이스 선택 시 가장 먼저 고려해야 할 점은 무엇인가요?**
  - **A.** 데이터의 특성과 서비스의 요구사항을 먼저 분석해야 합니다.
    - **데이터 모델**: 사용자 프로필, 게시글, 친구 관계 등 데이터 간의 관계가 복잡한가, 아니면 독립적인가?
    - **확장성(Scalability)**: 사용자가 급증할 때 서버를 쉽게 증설(수평 확장)할 수 있어야 하는가?
    - **일관성(Consistency)**: 친구 관계나 '좋아요' 수가 모든 사용자에게 항상 동일하게 보여야 하는가, 아니면 약간의 지연이 허용되는가?

### 2. 오개념 바로잡기 (Common Misconceptions)

- **오개념 1: "데이터베이스는 엑셀(Excel)과 비슷하다."**
  - **바로잡기**: 엑셀은 개인용 분석이나 소규모 데이터 관리에 유용한 도구이지만, 데이터베이스는 여러 사용자가 동시에 접근하여 대용량의 데이터를 안전하고 일관성 있게 관리하기 위한 전문 시스템입니다. DBMS는 동시성 제어, 보안, 데이터 무결성, 백업 및 복구 등 엑셀에는 없는 강력한 관리 기능을 제공합니다.

- **오개념 2: "데이터 분석은 무조건 복잡한 기술(AI, 머신러닝)을 사용해야 한다."**
  - **바로잡기**: 성공적인 분석은 기술이 아닌 '올바른 질문'에서 시작됩니다. '5 Whys' 기법처럼 문제의 근본 원인을 파고들거나, 비즈니스 목표를 측정 가능한 KPI로 정의하는 과정이 기술 적용보다 선행되어야 합니다. 간단한 SQL 쿼리나 엑셀 분석만으로도 충분히 가치 있는 인사이트를 얻는 경우가 많습니다.

- **오개념 3: "웹 스크래핑은 불법이다."**
  - **바로잡기**: 웹 스크래핑 자체는 기술적으로 중립적입니다. 하지만 모든 웹사이트가 스크래핑을 허용하는 것은 아닙니다. 사이트의 `robots.txt` 파일을 확인하여 크롤러의 접근 정책을 따라야 하며, 로그인 정보나 개인정보를 무단으로 수집하거나, 과도한 요청으로 서버에 부담을 주는 행위는 법적/윤리적 문제를 야기할 수 있습니다.
