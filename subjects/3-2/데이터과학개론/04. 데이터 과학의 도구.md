## 📝 요약 (Summary)
1.  **빅데이터(Big Data)**: 기존 데이터 처리 방식으로는 다루기 어려운 방대한 양의 정형, 비정형, 반정형 데이터를 의미합니다. 빅데이터의 핵심 특징인 **3V (Volume, Velocity, Variety)**에 **Value(가치)**와 **Veracity(정확성)**를 더해 **5V**로 설명합니다.
2.  **빅데이터 분석**: 데이터 마이닝, 기계 학습, 자연어 처리, 데이터 시각화 등 다양한 기술을 활용하여 데이터에서 인사이트를 발견하고 미래를 예측하며, 이를 통해 비즈니스 가치를 창출하는 과정입니다.
3.  **파이썬(Python)**: 문법이 간결하고 배우기 쉬우며, Pandas, Scikit-learn 등 강력한 라이브러리 생태계를 바탕으로 데이터 과학 전반에서 가장 널리 사용되는 범용 프로그래밍 언어입니다.
4.  **R**: 통계 분석과 데이터 시각화에 특화된 언어로, `ggplot2`와 같은 뛰어난 시각화 도구와 최신 통계 기법을 바탕으로 학술 및 연구 분야에서 강점을 보입니다.
5.  **Git & GitHub**: **Git**은 코드의 변경 이력을 관리하는 분산 버전 관리 시스템(DVCS)이며, **GitHub**는 Git으로 관리되는 프로젝트를 원격으로 저장하고 공유하며 협업할 수 있도록 돕는 웹 호스팅 서비스입니다.

### 🧐 해설

<a id="prob-1"></a>
**1. 다음 중 빅데이터의 초기 핵심 특징인 3V에 해당하지 않는 것은?**
> **정답: ③ 가치 (Value)**
> **해설**: 빅데이터의 초기 핵심 특징은 **규모(Volume)**, **속도(Velocity)**, **다양성(Variety)**의 3V였습니다. 이후 데이터의 **정확성(Veracity)**과 **가치(Value)**가 더해져 5V로 확장되었습니다. 따라서 '가치(Value)'는 5V에는 포함되지만 초기 3V에는 해당하지 않습니다.

<a id="prob-2"></a>
**2. 빅데이터 분석 방법 중, '앞으로 무슨 일이 일어날 것인가?'를 예측하는 데 중점을 두는 분석은 무엇인가?**
> **정답: ③ 예측 분석 (Predictive Analytics)**
> **해설**: 데이터 분석은 분석 수준에 따라 4가지 유형으로 나뉩니다.
> - **서술적 분석**: '무슨 일이 있었나?'를 파악합니다. (예: 월별 매출 보고서)
> - **진단적 분석**: '왜 일어났나?'를 분석합니다. (예: 특정 지역 매출 감소 원인 분석)
> - **예측 분석**: '무슨 일이 일어날까?'를 예측합니다. (예: 다음 분기 수요 예측)
> - **처방적 분석**: '무엇을 해야 하나?'를 제시합니다. (예: 예측된 수요에 맞춘 최적 재고 관리 방안)

<a id="prob-3"></a>
**3. 다음 중 파이썬(Python) 언어의 특징으로 가장 거리가 먼 것은?**
> **정답: ③ 변수 선언 시 자료형을 반드시 명시해야 하는 정적 타이핑(Static Typing) 언어이다.**
> **해설**: 파이썬은 변수에 값이 할당될 때 자료형이 결정되는 **동적 타이핑(Dynamic Typing)** 언어입니다. 따라서 `a = 10`, `a = "hello"`와 같이 변수 선언 시 자료형을 명시할 필요가 없으며, 같은 변수에 다른 자료형의 값을 넣을 수 있습니다. C, Java와 같은 언어가 정적 타이핑 언어에 해당합니다.

<a id="prob-4"></a>
**4. 다음 중 통계 분석 및 데이터 시각화에 특화된 R 언어에 대한 설명으로 옳지 않은 것은?**
> **정답: ③ 다른 프로그래밍 언어와의 연동이 불가능한 폐쇄적인 환경을 가지고 있다.**
> **해설**: R은 `Rcpp` 패키지를 통해 C++ 코드를 통합하거나, `reticulate` 패키지를 통해 파이썬 코드를 호출하는 등 다른 프로그래밍 언어와 활발하게 연동할 수 있는 개방적인 환경을 제공합니다.

<a id="prob-5"></a>
**5. 여러 개발자가 각자 자신의 컴퓨터에서 전체 저장소의 복제본을 가지고 작업하며, 오프라인에서도 코드 변경 이력을 관리할 수 있도록 하는 시스템은 무엇인가?**
> **정답: ② 분산 버전 관리 시스템 (DVCS)**
> **해설**: **분산 버전 관리 시스템(Distributed Version Control System, DVCS)**은 중앙 서버의 저장소와는 별개로, 모든 개발자가 자신의 컴퓨터에 전체 변경 내역을 포함한 완전한 저장소를 가집니다. 이로 인해 오프라인 상태에서도 커밋(변경 이력 저장)이 가능하며, 중앙 서버에 문제가 생겨도 각 개발자의 로컬 저장소를 통해 복구할 수 있습니다. Git이 가장 대표적인 DVCS입니다. 반면, 중앙 집중식 버전 관리 시스템(CVCS, 예: SVN)은 모든 변경 내역을 중앙 서버에서만 관리합니다.

<a id="prob-6"></a>
**6. 다음 중 분산 버전 관리 시스템인 Git과 이를 위한 웹 기반 호스팅 서비스의 가장 대표적인 조합은?**
> **정답: ③ Git & GitHub**
> **해설**: **Git**은 코드의 버전(이력)을 관리하는 프로그램(소프트웨어)이며, **GitHub**는 Git으로 관리하는 프로젝트를 올려두고 다른 사람과 공유하며 협업할 수 있는 웹 사이트(서비스)입니다. 이 둘은 데이터 과학 및 소프트웨어 개발 프로젝트에서 협업과 코드 관리를 위한 표준적인 조합으로 널리 사용됩니다.
> - ① Python은 프로그래밍 언어, Jupyter는 개발 환경입니다.
> - ② R은 프로그래밍 언어, RStudio는 R을 위한 통합 개발 환경(IDE)입니다.
> - ④ SVN은 중앙 집중식 버전 관리 시스템입니다.

## 🌐 최신 동향 및 추가 정보

데이터 과학 분야는 인공지능(AI) 기술의 발전과 함께 빠르게 변화하고 있습니다. 학부생으로서 알아두면 좋을 최신 도구 및 기술 동향은 다음과 같습니다.

### 1. 생성형 AI (Generative AI)와 데이터 과학
- **코딩 보조**: ChatGPT, GitHub Copilot과 같은 생성형 AI 도구는 데이터 분석에 필요한 코드를 자동으로 생성하거나, 복잡한 코드에 대한 해설을 제공하고, 오류를 수정하는 데 활용되어 개발 생산성을 크게 향상시키고 있습니다.
- **분석 및 시각화**: "X와 Y의 상관관계를 보여주는 파이썬 코드를 작성해줘" 와 같이 자연어로 요청하여 데이터 시각화 코드를 얻거나, 분석 결과 보고서의 초안을 작성하는 등 분석 과정 전반에서 보조 도구로 사용이 확대되고 있습니다.

### 2. AutoML (자동화된 머신러닝)
- **개념**: 전문가의 개입을 최소화하면서 머신러닝 모델 개발의 전체 파이프라인(데이터 전처리, 특징 공학, 모델 선택 및 하이퍼파라미터 튜닝)을 자동화하는 기술입니다.
- **의의**: 이를 통해 머신러닝에 대한 깊은 전문 지식이 없는 사람도 손쉽게 모델을 구축하고 실험할 수 있게 되어, 데이터 과학의 민주화에 기여하고 있습니다. Google Cloud AutoML, H2O.ai 등이 대표적인 플랫폼입니다.

### 3. MLOps (Machine Learning Operations)
- **개념**: 머신러닝 모델을 안정적이고 효율적으로 개발하고, 배포하며, 운영하기 위한 원칙과 기술의 집합입니다. 기존 소프트웨어 개발의 DevOps(개발+운영) 문화를 머신러닝 시스템에 적용한 것입니다.
- **중요성**: 단순히 모델을 한 번 개발하는 것을 넘어, 지속적으로 성능을 모니터링하고, 새로운 데이터로 모델을 업데이트하며, 전체 서비스 파이프라인을 안정적으로 관리하는 것의 중요성이 커지고 있습니다.

### 4. 증강 분석 (Augmented Analytics)
- **개념**: 데이터 분석 과정에 머신러닝과 자연어 처리(NLP) 같은 AI 기술을 접목하여, 데이터 준비부터 인사이트 도출, 설명에 이르는 과정을 자동화하고 향상시키는 접근 방식입니다.
- **활용**: 사용자가 복잡한 쿼리나 코딩 없이도 자연어로 질문하면 시스템이 알아서 데이터를 분석하고 관련 인사이트를 그래프나 차트와 함께 제시해주는 등, 비전문가도 쉽게 데이터를 탐색하고 이해할 수 있도록 돕습니다.

## 📚 핵심 용어집 (Glossary)

- **빅데이터 (Big Data)**: 기존의 데이터 처리 능력으로는 처리하기 어려운 방대한 규모, 빠른 생성 속도, 다양한 형태를 가진 데이터 또는 그 데이터를 처리하고 분석하는 기술을 의미합니다.

- **정형/비정형/반정형 데이터 (Structured/Unstructured/Semi-structured Data)**:
    - **정형 데이터**: 미리 정해진 구조와 형식에 따라 저장된 데이터입니다. (예: 관계형 데이터베이스의 테이블, 엑셀 시트)
    - **비정형 데이터**: 구조나 형식이 정해져 있지 않은 데이터입니다. (예: 텍스트 문서, 이미지, 동영상, 음성 파일)
    - **반정형 데이터**: 데이터 내에 스키마나 메타데이터를 포함하여 어느 정도의 구조를 가진 데이터입니다. (예: JSON, XML)

- **데이터 전처리 (Data Preprocessing)**: 수집된 원시 데이터를 분석에 적합한 형태로 가공하고 정제하는 과정입니다. 결측치 처리, 이상치 제거, 데이터 변환 등의 작업을 포함하며 분석 품질에 큰 영향을 미칩니다.

- **탐색적 데이터 분석 (EDA, Exploratory Data Analysis)**: 본격적인 데이터 분석에 앞서, 데이터를 다양한 각도에서 관찰하고 시각화하며 데이터의 전체적인 구조와 특징, 잠재적인 문제점 등을 파악하는 과정입니다.

- **파이썬 (Python)**: 간결한 문법과 풍부한 라이브러리를 바탕으로 데이터 과학, 인공지능, 웹 개발 등 넓은 분야에서 사용되는 인기 있는 프로그래밍 언어입니다.

- **R**: 통계 분석과 데이터 시각화에 특화된 프로그래밍 언어로, 특히 학술 및 연구 분야에서 널리 사용됩니다.

- **SQL (Structured Query Language)**: 관계형 데이터베이스(RDB)에서 데이터를 검색, 삽입, 수정, 삭제하는 등 데이터를 관리하고 조작하기 위한 표준 질의 언어입니다.

- **Pandas**: 파이썬에서 표 형태의 데이터를 쉽게 다룰 수 있도록 `DataFrame`이라는 자료구조를 제공하는 데이터 분석 핵심 라이브러리입니다.

- **NumPy**: 파이썬에서 다차원 배열과 행렬 연산을 효율적으로 수행할 수 있도록 지원하는 수치 계산용 라이브러리입니다.

- **Scikit-learn**: 파이썬을 위한 대표적인 머신러닝 라이브러리로, 분류, 회귀, 군집화 등 다양한 머신러닝 알고리즘과 모델 평가 도구를 제공합니다.

- **Jupyter Notebook**: 코드, 실행 결과, 설명, 시각화를 하나의 문서로 통합하여 관리할 수 있는 대화형 웹 기반 개발 환경입니다. 데이터 분석 과정을 기록하고 공유하는 데 매우 유용합니다.

- **Git**: 컴퓨터 파일의 변경사항을 추적하고 여러 사용자들 간에 해당 파일들의 작업을 조율하기 위한 분산 버전 관리 시스템(DVCS)입니다.

- **GitHub**: Git을 사용하는 프로젝트를 위한 웹 기반 호스팅 서비스로, 전 세계 개발자들과의 코드 공유 및 협업을 위한 핵심 플랫폼입니다.

### 1. 가상 시나리오: "데이터 분석 신입사원의 첫 프로젝트"

> **상황**: 당신은 IT 스타트업에 막 입사한 데이터 분석가입니다. 팀장님이 "우리 앱 사용자들의 이탈 원인을 분석해서 보고해주세요"라는 첫 과제를 주었습니다. 사용할 수 있는 데이터는 사용자 활동 로그(JSON 형식)와 가입 정보(사내 DB에 SQL로 저장)입니다.

**Q&A 기반 문제 해결 과정**
- **Q1. 어떤 도구부터 사용해야 할까요?**
  - **A.** 먼저 **SQL**을 사용하여 DB에서 필요한 사용자 가입 정보를 추출합니다. 그 다음, **Python**과 **Pandas** 라이브러리를 사용하여 JSON 형식의 활동 로그를 불러와 SQL로 추출한 데이터와 합칩니다.

- **Q2. 분석 과정은 어떤 환경에서 진행하는 것이 효율적일까요?**
  - **A.** **Jupyter Notebook** 환경을 추천합니다. 코드를 셀 단위로 실행하고 시각화 결과를 바로 확인할 수 있어, 데이터를 탐색하고(EDA) 분석 과정을 기록하기에 매우 편리합니다.

- **Q3. 분석이 끝난 후 코드는 어떻게 관리해야 할까요?**
  - **A.** 분석에 사용된 모든 코드(.ipynb, .sql)를 **Git**으로 버전 관리하고, **GitHub**의 개인 저장소에 올려 포트폴리오로 관리하는 것이 좋습니다. 이를 통해 작업 이력을 남기고, 나중에 다른 팀원과 협업하거나 코드 리뷰를 받을 때 유용하게 사용할 수 있습니다.

### 2. 오개념 바로잡기 (Common Misconceptions)

- **오개념 1: "데이터 과학을 하려면 Python과 R을 둘 다 마스터해야 한다."**
  - **바로잡기**: 둘 다 훌륭한 도구이지만, 대부분의 경우 하나를 깊이 있게 파는 것이 더 효율적입니다. Python은 범용성과 확장성 덕분에 산업계에서 더 폭넓게 사용되는 추세이며, R은 통계 분석과 시각화가 중요한 학계 및 연구 분야에서 강점을 보입니다. 자신의 목표와 주로 활동할 분야에 맞춰 주력 언어를 선택하고, 필요에 따라 다른 언어를 학습하는 것이 현명한 전략입니다.

- **오개념 2: "Git과 GitHub는 개발자들만 사용하는 어려운 도구이다."**
  - **바로잡기**: Git과 GitHub는 이제 데이터 과학자에게도 필수적인 협업 및 포트폴리오 관리 도구입니다. 분석 코드, 쿼리, 결과물 등을 체계적으로 관리하고, 동료와 함께 프로젝트를 진행하며, 자신의 분석 역량을 외부에 보여주는 '이력서'로 활용할 수 있습니다. 최근에는 GUI(그래픽 사용자 인터페이스) 도구도 많아져 과거보다 훨씬 쉽게 배울 수 있습니다.

- **오개념 3: "SQL은 오래된 기술이라 더 이상 중요하지 않다."**
  - **바로잡기**: 데이터 과학의 시대에 SQL의 중요성은 오히려 더 커졌습니다. 대부분의 기업 데이터는 여전히 관계형 데이터베이스에 저장되어 있으며, 분석을 위해 이 데이터에 접근하고 필요한 형태로 가공하는 첫 단계가 바로 SQL입니다. Pandas와 같은 도구로 모든 원본 데이터를 가져와 처리하는 것보다, SQL을 통해 서버에서 필요한 데이터만 효율적으로 추출하는 것이 훨씬 빠르고 효과적입니다.
