## Page 1: 3개월 학습 로드맵 및 접근법

성공적인 데이터 과학 학습의 첫걸음은 문제 해결 프로세스를 이해하고, 데이터를 다루는 기술을 익히는 것입니다. 아래 로드맵은 방대한 데이터 과학의 세계를 체계적으로 탐험할 수 있도록 안내합니다.

### **1개월 차: 데이터 과학적 사고방식과 친해지기**

*   **목표:** 데이터 과학이 무엇인지 이해하고, 데이터를 통해 문제를 정의하고 해결하는 사고방식을 익힙니다.
*   **학습 내용:**
    *   **데이터 과학이란?**: 정의, 중요성, 그리고 컴퓨터 과학, 통계학, 도메인 지식과의 관계를 이해합니다.
    *   **데이터 과학 프로젝트 생애주기**: 문제 정의 → 데이터 수집 → 데이터 정제 → 탐색적 데이터 분석(EDA) → 모델링 → 시각화 및 보고의 전체 흐름을 파악합니다.
    *   **기초 통계 복습**: 기술통계(평균, 중앙값, 최빈값, 분산, 표준편차)와 추론통계(가설검정, 상관관계)의 기본 개념을 다시 확인합니다.
*   **학습 방법:**
    *   데이터 과학 입문 교양서적(Page 3 참고)을 읽으며 큰 그림을 그립니다.
    *   Kaggle, 기업 기술 블로그 등에서 실제 데이터 분석 사례를 찾아보며 데이터로 어떤 문제를 해결할 수 있는지 감을 잡습니다.

### **2개월 차: 데이터 분석을 위한 파이썬 라이브러리 정복**

*   **목표:** 데이터 분석의 핵심 도구인 `NumPy`, `Pandas`, `Matplotlib` 사용법을 능숙하게 익힙니다.
*   **학습 내용:**
    *   **NumPy**: 다차원 배열 `ndarray`를 이해하고, 벡터화 연산(Vectorized Operation)을 통해 빠른 수치 계산을 수행하는 방법을 학습합니다.
    *   **Pandas**: `Series`와 `DataFrame`을 중심으로 데이터를 불러오고(CSV, Excel), 원하는 형태로 정제하며(결측치, 중복값 처리), 데이터를 선택, 필터링, 정렬, 그룹화하는 방법을 마스터합니다.
    *   **Matplotlib & Seaborn**: 데이터를 시각화하여 인사이트를 발견하는 방법을 배웁니다. 선 그래프, 막대 그래프, 히스토그램, 산점도 등 기본 차트를 자유자재로 그리는 연습을 합니다.
*   **학습 방법:**
    *   온라인 튜토리얼이나 강의(Page 3 참고)를 따라 Jupyter Notebook에서 직접 코드를 실행하며 학습합니다.
    *   Kaggle의 'Titanic', 'Iris'와 같은 기본 데이터셋을 활용하여 나만의 탐색적 데이터 분석(EDA) 노트를 만들어봅니다.

### **3개월 차: 머신러닝 핵심 개념 맛보기**

*   **목표:** 머신러닝의 기본 원리를 이해하고, `Scikit-learn` 라이브러리를 통해 간단한 모델을 구축하고 평가하는 과정을 경험합니다.
*   **학습 내용:**
    *   **머신러닝의 종류**: 지도학습(Supervised), 비지도학습(Unsupervised), 강화학습(Reinforcement)의 차이를 이해합니다.
    *   **지도학습**:
        *   **회귀(Regression)**: 선형 회귀(Linear Regression) 모델을 통해 연속적인 값을 예측하는 원리를 학습합니다.
        *   **분류(Classification)**: k-최근접 이웃(k-NN), 결정 트리(Decision Tree) 모델을 통해 데이터가 어떤 범주에 속하는지 예측하는 원리를 학습합니다.
    *   **모델 평가**: 훈련(Train) 데이터와 테스트(Test) 데이터 분리의 중요성을 이해하고, 정확도(Accuracy) 등 기본 모델 평가 지표를 학습합니다.
    *   **비지도학습**: K-평균(K-Means) 알고리즘을 통해 데이터 군집화(Clustering)의 개념을 맛봅니다.
*   **학습 방법:**
    *   `Scikit-learn` 공식 문서를 보며 예제 코드를 따라 쳐봅니다.
    *   수학적 깊이보다는 '어떤 문제를 어떤 모델로 풀 수 있는가'에 대한 직관을 기르는 데 집중합니다.

## Page 2: 반드시 알아야 할 핵심 학습 내용

수업 시작 전 아래 개념들을 확실히 이해해두면, 강의 내용을 훨씬 수월하게 따라갈 수 있습니다.

### **Part 1. 데이터 분석의 핵심 프로세스**

*   **탐색적 데이터 분석 (EDA, Exploratory Data Analysis):** 본격적인 모델링에 앞서 데이터를 다양한 각도에서 관찰하고 이해하는 과정. 시각화와 통계치를 통해 데이터의 패턴, 이상치, 변수 간 관계를 파악하는 단계로, 데이터 분석의 가장 중요한 과정 중 하나입니다.
*   **데이터 전처리 (Data Preprocessing):** 원시 데이터(Raw Data)를 분석에 적합한 형태로 가공하는 과정. 결측치 처리, 이상치 제거, 데이터 정규화(Normalization), 범주형 데이터 인코딩 등이 포함됩니다.
*   **특성 공학 (Feature Engineering):** 기존 변수(Feature)를 조합하거나 변형하여 모델의 성능을 높일 수 있는 새로운 변수를 만드는 과정. 도메인 지식이 매우 중요하게 작용합니다.

### **Part 2. 파이썬 핵심 라이브러리**

*   **NumPy (Numerical Python):** 과학 계산을 위한 핵심 라이브러리. `ndarray`라는 효율적인 다차원 배열 객체를 제공하여 빠르고 강력한 수치 연산을 가능하게 합니다.
*   **Pandas:** 행과 열로 이루어진 테이블 형태의 데이터를 다루는 데 특화된 라이브러리. 데이터 조작, 정제, 분석을 위한 강력하고 유연한 자료구조(`DataFrame`)를 제공합니다.
*   **Matplotlib / Seaborn:** 데이터 시각화를 위한 라이브러리. Matplotlib은 기본적인 차트를, Seaborn은 더 아름답고 통계적인 차트를 쉽게 그릴 수 있도록 도와줍니다.
*   **Scikit-learn:** 파이썬 최고의 머신러닝 라이브러리. 다양한 분류, 회귀, 군집화 알고리즘과 모델 평가, 데이터 전처리 도구를 포함하고 있습니다.

### **Part 3. 머신러닝 기초 개념**

*   **지도학습 vs 비지도학습 (Supervised vs Unsupervised Learning):** 정답(Label)이 있는 데이터로 모델을 학습시키는지(지도), 정답 없이 데이터 자체의 구조를 학습하는지(비지도)의 차이.
*   **회귀 vs 분류 (Regression vs Classification):** 주가, 온도처럼 연속적인 값을 예측하는 문제(회귀)와 스팸메일 여부, 고객 등급처럼 정해진 범주 중 하나를 예측하는 문제(분류)의 차이.
*   **과적합 (Overfitting):** 모델이 훈련 데이터에만 너무 치중하여 학습한 나머지, 새로운 데이터에 대한 예측 성능이 떨어지는 현상. 모델의 일반화 능력이 부족한 상태를 의미합니다.
*   **훈련/검증/테스트 세트 (Train/Validation/Test Set):** 모델을 학습시키기 위한 데이터(훈련), 모델의 성능을 조정(튜닝)하기 위한 데이터(검증), 최종 모델의 성능을 평가하기 위한 데이터(테스트)로 데이터를 분리하여 사용합니다.

## Page 4: 시험 핵심 개념 (데이터 분석 심화)

아래는 시험에 단골로 출제되며, 실무에서도 매우 중요한 데이터 분석의 핵심 개념입니다. 원리를 정확히 이해하고 코드로 구현하는 연습이 필요합니다.

### **1. 탐색적 데이터 분석 (Exploratory Data Analysis, EDA)**

*   **키워드:** `데이터 정제`, `결측치 처리`, `이상치 탐지`, `시각화`, `상관 분석`

*   **상세 설명:**
    EDA는 데이터에 대한 '첫인상'을 파악하고, 데이터가 가진 구조와 패턴, 잠재적인 문제점을 발견하는 과정입니다. EDA를 통해 데이터의 품질을 확인하고, 어떤 전처리나 특성 공학이 필요할지, 어떤 모델이 적합할지에 대한 힌트를 얻을 수 있습니다.

    *   **기술 통계 확인:** `pandas.DataFrame.describe()`로 각 변수의 평균, 표준편차, 사분위수 등을 확인하여 데이터의 분포를 파악합니다.
    *   **결측치(Missing Value) 처리:** `isnull().sum()`으로 결측치를 확인하고, 이를 특정 값으로 채우거나(Imputation), 해당 데이터를 제거하는 전략을 수립합니다.
    *   **시각화를 통한 인사이트 발견:**
        *   **히스토그램/커널 밀도 추정:** 단일 변수의 분포 형태 확인.
        *   **박스 플롯(Box Plot):** 변수의 분포, 이상치(Outlier) 탐지.
        *   **산점도(Scatter Plot):** 두 연속형 변수 간의 관계 확인.
        *   **상관관계 히트맵(Correlation Heatmap):** 여러 변수 간의 상관관계를 한눈에 파악.

*   **추가 학습자료:**
    *   **Kaggle Notebook:** 'Titanic' 데이터셋에 대한 상위권 EDA 노트북을 여러 개 필사해보는 것이 가장 좋은 학습법입니다.
    *   **도서:** '파이썬 라이브러리를 활용한 데이터 분석' 4장(NumPy), 5장(Pandas)

### **2. 선형 회귀 (Linear Regression)**

*   **키워드:** `지도학습`, `회귀`, `가설 함수`, `비용 함수(Cost Function)`, `경사 하강법(Gradient Descent)`, `결정 계수(R-squared)`

*   **상세 설명:**
    선형 회귀는 하나 이상의 독립 변수(X)를 사용하여 종속 변수(y)의 값을 예측하는 가장 기본적인 회귀 알고리즘입니다. 데이터에 가장 잘 맞는 '직선'을 찾는 것이 목표입니다.

    *   **가설 함수 (Hypothesis Function):** `H(x) = Wx + b` (W: 가중치, b: 편향). 우리가 찾으려는 예측 모델(직선)입니다.
    *   **비용 함수 (Cost Function):** 실제 값과 모델의 예측 값 사이의 오차(잔차)를 측정하는 함수. 보통 평균 제곱 오차(MSE, Mean Squared Error)를 사용합니다. `cost = mean((H(x) - y)^2)`
    *   **경사 하강법 (Gradient Descent):** 비용 함수를 최소화하는 가중치(W)와 편향(b)을 찾는 최적화 알고리즘. 비용 함수의 기울기(경사)가 가장 낮은 방향으로 파라미터를 점진적으로 업데이트합니다.
    *   **모델 평가:**
        *   **MSE (Mean Squared Error):** 오차의 제곱 평균. 작을수록 좋습니다.
        *   **결정 계수 (R²):** 모델이 데이터의 분산을 얼마나 잘 설명하는지를 나타내는 지표. 0과 1 사이의 값을 가지며, 1에 가까울수록 모델의 설명력이 높음을 의미합니다.

*   **추가 학습자료:**
    *   **유튜브:** StatQuest의 'Linear Regression' 영상, Andrew Ng의 'Machine Learning' 강의 1~2주차
    *   **Scikit-learn:** `sklearn.linear_model.LinearRegression` 공식 문서의 예제 코드 실습
